Gerrit初始化代码并同步

  • Gerrit

#Gerrit初始化代码并同步
建立好Gerrit账户以后，就可以同步代码了。我们使用Repo和Git来进行项目和代码的版本管理。xxxx的代码由数个子项目构成，每个子项目隶属于一个GIT项目。在这些子项目上面是通过repo来管理的，类似于Android源代码的管理方式。

  • 下载drepo命令文件至某个PATH的目录chmod a+x ~/bin/drepo
  • 初始化工程repo init -u ssh://<username>@codeserver.xxx.com:29418/BrowserManifest -b <branch_name>

最后的branch_name代表一个repo branch，是我们的开发版本，比如shell_master，xxx_master等，请和对应的项目负责人确认具体的 repo branch

  • 同步代码drepo sync
  • 创建本地分支（这一步是必须的，否则每次同步都会冲掉本地的提交）drepo start <branch name> (brance name 任意，推荐和上面-b参数指定的branch name一致)
    repo list 可以查看repo环境下有多少个project
  • 如出现错误：error: at least one project must be specified
    运行
    drepo start <branch name> (brance name 任意，推荐和上面-b参数指定的branch name一致) --all

然后其他操作跟git差不多

但是最后上传的时候不同
repo upload
